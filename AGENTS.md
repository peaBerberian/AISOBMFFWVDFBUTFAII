# Repository Guidelines

## Project Shape

This is a small static browser app for inspecting ISOBMFF/MP4 files. There is
no server-side application: `build/index.html` loads `build/style.css` and the
browser bundle generated from `src/index.js`.

Parsing is provided by the `isobmff-inspector` dependency. This repository owns
the UI, streaming/input handling, rendering, and presentation around that
parser. Avoid adding parser logic here unless it is strictly UI-facing.

## Important Files

- `src/index.js`: app entry point. Initializes local file input, URL input, tab
  navigation, GitHub stars, abort handling, and loading state.
- `src/parse.js`: connects `isobmff-inspector` streaming parse events to the UI.
  It incrementally renders the box tree and then renders the size chart.
- `src/ProgressBar.js`: progress/status/cancel UI state.
- `src/utils.js`: stream helpers and abort-aware async iteration.
- `src/tabs/tree.js`: box tree DOM rendering.
- `src/tabs/sizes.js`: size chart/map/table rendering.
- `src/tabs/utils.js`: small DOM/formatting helpers used by tab renderers.
- `src/styles/*.css`: authored CSS source files.
- `build/index.html`: static page shell.
- `build/style.css`: generated by the build script from `src/styles/*.css`.
- `build/aisobmffwvdfbutfaii.js`: generated browser bundle.
- `build/aisobmffwvdfbutfaii.min.js`: generated minified browser bundle.

## Commands

- `npm run build`: concatenates CSS sources into `build/style.css` and builds
  `build/aisobmffwvdfbutfaii.js`.
- `npm run min`: same CSS build, plus a minified JS bundle at
  `build/aisobmffwvdfbutfaii.min.js`.
- `npm run watch`: rebuilds JS through esbuild watch mode and rebuilds
  `build/style.css` when `src/styles/*.css` changes.
- `npm run check`: runs Biome checks and TypeScript checking.
- `npm run format`: applies Biome formatting.

Run `npm run check` before handing off code changes.

## CSS Organization

CSS is split by UI ownership, not by generic CSS category:

- `base.css`: variables, reset, body defaults.
- `progress.css`: progress bar, status line, cancel button.
- `shell.css`: page header, file/URL selection, result loading state, tabs.
- `box-tree.css`: box tree structure and box headers.
- `values.css`: parsed value tables, value formatting, tooltips, flags, issues.
- `sizes.css`: size summary, map, sortable rows, and related responsive rules.

Prefer adding styles to the file that owns the rendered UI. If a new feature is
large enough to have its own rendering module, it is usually large enough to
have its own CSS file.

When adding or removing a CSS source file, update the `styleSources` list in
`scripts/build.mjs`. The explicit list is the source order.

Use existing CSS variables before adding new colors or spacing constants. Keep
selectors close to the DOM classes emitted by the rendering module. Avoid broad
global selectors except in `base.css`.

## JavaScript Conventions

The codebase uses browser-native DOM APIs and ES modules bundled by esbuild.
Stay with that style; do not introduce a framework for local UI work.

Use JSDoc types for non-obvious data shapes and imported parser types. This is
how the project gets useful TypeScript checking while staying in JavaScript.

Preserve abort behavior. File and URL parsing are cancelable through a single
current `AbortController`; long-running or streaming work should observe the
provided `AbortSignal`.

Rendering code generally builds DOM nodes directly. When inserting parser data,
prefer `textContent` or the existing `esc()` helper over raw `innerHTML`.

The app intentionally yields during stream consumption in `src/utils.js` so the
browser can keep painting during heavy parsing. Be careful with changes that
turn streaming work into one large synchronous render.

## UI Behavior Notes

Parsing starts on local file selection or URL submit. Starting a new parse
aborts the previous one, dims stale results, resets to the box tree tab, and
shows progress. `parseAndRender` reveals tabs only once rendering can display
real results.

The box tree is rendered incrementally from parser events. Top-level boxes are
also accumulated for the size chart. The size chart renders after parsing
finishes, using the final parsed box tree.

The size view treats `mdat` as excluded for the "excluding mdat" scale. Keep
that behavior explicit if changing size calculations or labels.

## Dependency Boundaries

The parser package is the source of truth for ISOBMFF parsing. UI code may
format, filter, visualize, and annotate parsed data, but should not duplicate
box parsing behavior that belongs in `isobmff-inspector`.

Network fetches are browser fetches from user-provided URLs. Keep the app
static-hosting compatible and client-only.
